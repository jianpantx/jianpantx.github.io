vagrant+homestead+phpstorm+xdebug调试laravel

2019-04-20

🐱‍👤Xdebug是php的调试神器，之前也安装使用过，但是本地开发环境是phpstudy。现在基本上都在用vagrant+homestead来本地开发，所以要重新搭。在安装过程中，也是踩了好几个坑，装好的时候心里是奔溃的...

Xdebug安装

  1. 首先用git拉取源码安装
git clone https://github.com/xdebug/xdebug.git
  1. 进入目录，切换到最新版本的分支，这个可以在gitub上看最新的分支然后切换，我这里最新的是Xdebug_2_7
git checkout  xdebug_2_7

3.接下来有两种安装方式，一种是默认安装，另一种是自定义安装，我这里选择的是默认安装，执行目录里的安装脚本即可：

./rebuild.sh

安装完成之后会返回一个文件目录，里面有一个后面配置需要的文件xdebug.so 4.在php.ini中添加xdebug扩展 正常情况下实在在php.ini文件默认添加xdebug配置参数

zend_extension=xdebug.so    //这里填写正确的文件路径
xdebug.idekey = "vagrant"
xdebug.default_enable = 1
xdebug.remote_connect_back = 1
xdebug.remote_port = 9001
xdebug.remote_enable = 1
xdebug.remote_autostart = 1
xdebug.remote_handler="dbgp"

在homestead中，已经帮有我们需要的xdebug配置文件 只需在20-xdebug.ini修改对应参数即可，如果想添加在php.ini话，要把这个文件里的参数注释，不然会发生冲突（排这个坑花了好长时间😭）。
