laravel项目中提高phpstorm开发效率的扩展和插件

· 2019-04-18 · # php # laravel # phpstorm

🎈文章会列出一些在phpstorm开发中可以给我带来许多phpstorm插件和开发laravel的扩展，会不断补充和修改 下面是我自己phpstorm中使用的插件

1.Material Theme UI 界面美化 2.translation 翻译 3.rainbow brackets 使用不颜色来区分不同层级的括号 4.ideavim 想在打代码的时候使用vim模式的可以安装 5.key promoter X 提示当前一些操作的快捷键 方便记忆快捷键 . . . 未完待续

以下是laravel扩展

一、Laravel-ide-helper

laravel在不安装插件的情况下，phpstorm开发的代码提示和方法的跳转很不友好，会看到很多方法下面有很多波浪线，这对于有强迫症的人是坚决不能忍受的。安装Laravel-ide-helper这个扩展可以帮我们解决这些问题。

安装

在安装了composer的情况下，执行安装命令：

composer require barryvdh/laravel-ide-helper --dev

将扩展的服务提供者添加到config/app.php中的providers数组中

Barryvdh\LaravelIdeHelper\IdeHelperServiceProvider::class,

也可以在laravel中的register()中添加以下代码，而不需要在config/app.php添加

public function register()
{
    if ($this->app->environment() !== 'production') {
        $this->app->register(\Barryvdh\LaravelIdeHelper\IdeHelperServiceProvider::class);
    }
    // ...
}

facades生成phpDoc，在项目会生成一个_ide_helper.php文件

php artisan ide-helper:generate

这样就完成全部安装，prefect！👏

二、Laravel Debugbar

以前用tp开发的时候，都会用到调试工具栏，laravel怎么可能没有，而且比trace好看多了，哈哈哈~ 😎

使用composer安装：

composer require barryvdh/laravel-debugbar --dev

将服务提供者类添加到config/app.php中的provider数组

Barryvdh\Debugbar\ServiceProvider::class,

如果要使用Facade记录消息，请将其添加到app.php中的外观中：

'Debugbar' => Barryvdh\Debugbar\Facade::class,

最后，确定配置中APP_DEBUG是true，在浏览器访问项目就可以使用了
